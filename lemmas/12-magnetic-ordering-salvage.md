# Lemma 12 Draft: Magnetic-Ordering Salvage

## Question

- Status: Conjectural. If one insists on salvaging an electric-only minimal-sector theorem, what explicit magnetic-ordering assumption is actually strong enough?

## Salvage Options That Fail

- Status: Proven. No extra magnetic assumption means no salvage: `B2` enters already at `\Delta = 2`.
- Status: Proven. Mild one-step suppression also fails: if `B` is shifted upward by only one unit of fixed-order weight, then `B2` still enters at `\Delta = 4`.
- Status: Proven. Therefore neither of those choices rescues the electric-only theorem inside the current MVP window.

## Salvage Options That Work Conditionally

- Status: Proven. Strong ordering salvage works only if every scalar containing `B` is pushed above `\Delta = 4`; equivalently, `B` must be assigned weight `3` or higher, or an equivalent stronger suppression must be imposed.
- Status: Proven. Background-restriction salvage works only if the allowed theorem domain is explicitly restricted to backgrounds where `B_ij` vanishes or is negligible.
- Status: Proven. Matching-level salvage works only if one explicitly assumes vanishing or parametrically suppressed magnetic Wilson data for the body family under study.

## Cost Of Salvage

- Status: Proven. Strong ordering salvage adds a new structural assumption beyond the current theorem setup.
- Status: Proven. Background-restriction salvage turns the statement into a theorem about a restricted environment class rather than a general minimal sector.
- Status: Proven. Matching-level salvage turns the statement into a body-dependent conditional theorem rather than a structural closure theorem.

## Result

- Status: Proven. Electric-only salvage is possible only conditionally.
- Status: Proven. Any successful salvage requires a stronger explicit assumption than the current repository presently adopts.
- Status: Conjectural. For novelty, the `E/B`-expanded theorem route remains stronger than electric-only salvage because it enlarges the theorem rather than the assumption stack.
