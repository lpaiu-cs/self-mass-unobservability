# Family-Admission No-Go

## Exact Negative Claim

- Status: Proven. The current repository no longer treats minimal-sector uniqueness as the positive theorem target.
- Status: Proven. For every explicitly audited unsuppressed primitive-family enlargement so far, a new low-order survivor appears.
- Status: Proven. Therefore a stronger theorem that claims physical minimal-sector uniqueness is already obstructed unless explicit suppression, ordering, or background-restriction assumptions are added family by family.

## Audited Families

- Status: Proven. Magnetic tidal STF family `B_ij`: the smallest explicit new survivor is `B2`.
- Status: Proven. Scalar-like family with bare rank-0 source `S`: the smallest explicit new survivor is `S`.
- Status: Proven. Derivative-only or shift-symmetric scalar family: the first new survivors appear at weight `4`, and a canonical explicit obstruction is `dotS2`.

## Family Classification Step

- Status: Proven. Class A: if an admitted family carries its own parity-even scalar at weight `1` or `2`, minimal-sector uniqueness fails immediately.
- Status: Proven. The audited examples are `S` and `B2`.
- Status: Proven. Class B: even if shift symmetry removes the bare scalar source, derivative invariants can still survive at the same fixed truncation order.
- Status: Proven. The audited derivative-only scalar examples are `dotS2`, `gradS2`, `DtS_E2`, `DtS_B2`, and `divEGradS`.
- Status: Conjectural. Additional unsuppressed primitive families are expected to behave similarly unless an explicit reduction or ordering principle is written down in advance.

## What This Does And Does Not Kill

- Status: Proven. This no-go kills minimal-sector uniqueness as a default theorem target.
- Status: Proven. This no-go does not kill the broader finite-family fixed-order collapse program.
- Status: Proven. The negative result is about uniqueness of the physically justified minimal sector, not about existence of finite family-conditioned normal forms.
